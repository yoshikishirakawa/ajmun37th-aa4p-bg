# コラム1：Webフォントの歴史

## デジタルタイポグラフィの発展

Webにおける日本語表示技術の変遷を振り返ります。

---

## 画像フォントの時代

1990年代は以下の手法が主流でした：

- **GIF画像**: 文字を画像として埋め込む
- **テーブルレイアウト**: 細密な位置調整
- **固定ピクセル**: デバイス非依存性の欠如

### 課題

- 検索不能
- アクセシビリティの欠如
- 国際化の困難さ

---

## Webフォントの登場

1998年にCSS2で`@font-face`が標準化されました[^20_col01-1]。

### 日本語対応の課題

- **ファイルサイズ**: 数千文字の膨大なデータ量
- **ライセンス**: 商用利用の制限
- **ブラウザ互換性**: 実装のばらつき

WOFF2の登場により、ファイルサイズの問題は大幅に改善されました。

---

## サブセット化の重要性

現代では以下のアプローチが主流です：

### 1. 静的サブセット

```
必要な文字集合だけを抽出
├─ 基本漢字 2,135字
├─ ひらがな/カタカナ 140字
├─ 記号・英数字 200字
└─ カスタム文字集合
```

### 2. 動的サブセット

表示が必要になるたびにフォントデータを生成します。

---

## 現代のベストプラクティス

### フォント選定の基準

1. **ライセンスの明瞭性**
2. **可読性の高さ**
3. **ファイルサイズの適正化**
4. **ブラウザ互換性**

### 表示戦略

```css
@font-face {
  font-family: 'UDPMincho';
  src: url('../fonts/UDPMincho.woff2') format('woff2');
  font-display: swap;
  unicode-range: U+0000-30FF; /* 基本文字集合 */
}
```

---

## 今後の展望

Variable Fontsや最新の圧縮技術により、日本語Webフォントの可能性はさらに拡大しています。
