# 第1章 プロジェクトの概要

## 1.1 はじめに

本章ではプロジェクト全体の概要と目的について説明します。

## 1.2 プロジェクトの目的

### 1.2.1 課題認識

日本語の長文ドキュメントをWebで配信する際には、以下の課題が存在します：

1. **読書環境の多様性**
2. **文字体裁の維持**
3. **引用文献の管理**
4. **印刷と画面表示の両立**

### 1.2.2 解決方針

これらの課題に対し、以下のアプローチを採用します：

- **レスポンシブデザイン**
- **Webフォントの活用**
- **章ごとの文献管理**
- **複数の出力形式対応**

## 1.3 技術選定

### 1.3.1 Quarto + Pandoc

本研究では**Quarto**と**Pandoc**を中核技術として選択しました[^01_ch01-1]。

選定理由：
- 学術文書生成の標準ツール
- Markdownからの変換が容易
- PDF出力の高品質性
- プラグイン拡張性

### 1.3.2 UDP明朝フォント

本文書では**UDP明朝**を主要な日本語フォントとして使用します[^01_ch01-2]。

特長：
- 読みやすさ
- 印刷適性
- Webフォントとしての利用可能性

## 1.4 プロジェクト構成

### 1.4.1 ディレクトリ構造

```
project/
├─ index.md
├─ _quarto.yml
├─ content/
├─ src/
├─ filters/
├─ scripts/
├─ meta/
└─ build/
```

### 1.4.2 ファイル配置戦略

章ファイルは数値プリフィクスで順序を管理します[^01_ch01-3]。

- `00_front.md`: フロント挨拶
- `01_ch01.md`〜`07_ch07.md`: 本文
- `20_col01.md`〜`22_col03.md`: コラム
- `90_afterword.md`: 編集後記
